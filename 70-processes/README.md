# 70-processes

ITSM process designs — workflows CCH runs day-to-day. One subfolder per process: Incident, Problem, Change, Request, Knowledge, Configuration Management.

Configuration Management (`cm/`) is the crossroads of this vault: its notes link densely into [[10-csdm]] (the data model it governs), [[20-cch]] (the deployed CMDB), [[50-mappings]] (the actual mapping artefacts), and [[60-projects]] (integrations feeding the CMDB).
